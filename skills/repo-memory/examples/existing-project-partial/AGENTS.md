# AGENTS.md

This sample repo uses `docs/README.md` as the ownership map. Some useful legacy
docs remain canonical owners instead of being duplicated.

When starting work:

1. Read `docs/README.md`.
2. Follow the ownership map to the right owner.
3. Read `docs/feature-registry.md`.
4. Read the active feature doc before editing related code.

Keep this file thin. Useful legacy docs still exist in the repo, and current
mutable project state belongs in the owner named by `docs/README.md`.
