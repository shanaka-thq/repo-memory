# Generator Mode

Use this mode to generate derived indexes, registries, and work queues from canonical docs.

## Instructions

1. **Source data:** Parse the canonical feature files (`docs/features/*.md`) and doc files to extract status, priority, readiness, and metadata.
2. **Target destination:** Write output files to the configured generated directory (default: `docs/generated/`).
3. **No direct manual edits:** Never manually edit files in the generated path. They are derived and will be overwritten on generation.
4. **Header requirement:** Every generated file must start with the warning header:
   `<!-- Generated by repo-memory. Do not edit directly. -->`
5. **Deterministic output:** Ensure the generation sort order, layout, and styling are deterministic so they do not produce noisy Git diffs.
6. **Outputs to produce:**
   - `docs/generated/feature-registry.md`: Feature list summary table.
   - `docs/generated/next-work-queue.md`: Ranked work queue sorted by blocked state, readiness, priority, and date.
   - `docs/generated/doc-health.md`: Report containing verification dates, warnings, duplicates, and unmapped directories.
7. **Run CLI generation:** Run `npx repo-memory generate` to execute.
