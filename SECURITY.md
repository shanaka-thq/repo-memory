# Security Policy

## Supported Versions

This repository is a documentation skill, not a runtime service. Security fixes apply to the current released major version and the latest minor version in active development.

| Version | Supported |
| --- | --- |
| 2.x | Yes |
| 1.x | No, except for documentation-only clarification when needed |

## Reporting a Vulnerability

Open a private security advisory or contact the repository maintainer through the configured GitHub security reporting flow.

Relevant issues include malicious instructions, unsafe automation guidance, credential-handling mistakes, or documentation patterns that could cause agents to expose secrets, bypass review, or corrupt user repositories.

Expected handling:

- Reports should include the affected file, unsafe behavior, and a suggested safer wording when possible.
- Accepted issues should be fixed in the next patch or minor release depending on severity.
- If the issue is documentation quality rather than security, it may be redirected to a normal issue or pull request.
