# Evidence Extraction Example

This example shows how a Repo Memory agent can capture extracted behavior
without pretending the extraction report is canonical truth.

The evidence report stays in `docs/intake/`. Accepted findings are promoted
into mapped owners such as requirements, feature registry, feature docs, and
doc health.

## Files

- [`docs/intake/2026-05-22-evidence-extraction.md`](./docs/intake/2026-05-22-evidence-extraction.md) — raw evidence extraction report.
- [`docs/feature-registry.md`](./docs/feature-registry.md) — accepted feature candidate and next-work queue.
- [`docs/features/import-preview.md`](./docs/features/import-preview.md) — canonical feature handoff after promotion.
- [`docs/doc-health.md`](./docs/doc-health.md) — verification state and remaining question.
