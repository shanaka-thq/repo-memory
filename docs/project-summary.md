## 90-Second Orientation

Repo Memory is a documentation standard and skill for keeping durable project context in-repo so humans and coding agents can resume work safely.

The core idea is simple:

1. Keep one canonical owner per documentation capability.
2. Preserve useful existing docs rather than duplicating them.
3. Maintain feature state and handoff context in shared docs.
4. Validate docs structure and freshness as part of normal workflow.
